# RoboLane-X：视觉巡线与路口识别机器人系统

> 面向足式/人形教育机器人平台的轻量级视觉导航项目，支持白线巡线、路口交点识别与动作组联动控制。

## 一、项目简介

`RoboLane-X` 是一个基于 OpenCV 与机器人动作组控制的视觉导航系统。项目通过摄像头实时获取画面，对地面白线进行分区加权跟踪，并结合霍夫直线与交点计算实现路口检测，最终驱动机器人完成前进、微调转向与路口通过。

该项目适合作为：

- 机器人视觉方向的课程/竞赛作品
- 嵌入式 AI 与控制结合的工程化示例
- 面试中展示“感知-决策-执行”闭环能力的项目材料

## 二、核心功能

- 白线巡线：基于 LAB 色彩空间阈值分割目标线条
- 分区加权定位：将图像划分为上/中/下三段并加权计算中心
- 路口识别：通过边缘检测 + 概率霍夫直线检测候选线段
- 交点计算：对线段进行两两求交，识别有效路口点
- 动作联动：根据偏移量触发前进、左转、右转动作组

## 三、技术方案

### 1）视觉感知

- 图像预处理：缩放、Gaussian Blur、形态学腐蚀/膨胀
- 颜色分割：LAB 阈值（默认白色）
- 轮廓提取：寻找最大有效轮廓并计算最小外接矩形
- 中心估计：多 ROI 中心点按权重融合得到最终巡线中心

### 2）路径决策

- 横向误差：使用 `line_center_x - img_centerx` 作为控制依据
- 控制策略：
  - 偏差过大向右：执行 `turn_right_small_step`
  - 偏差过大向左：执行 `turn_left_small_step`
  - 偏差在可接受范围：执行 `go_forward_fast`

### 3）路口检测

- 使用 Canny 边缘提取有效边缘
- 使用 `HoughLinesP` 检测线段
- 通过斜率差与线段参数方程计算交点
- 检测到交点后触发直行/通过动作（可扩展为转向决策）

## 四、项目结构

当前版本为单文件核心脚本：

- `VisualPatrol224.py`：主程序（视觉处理、状态控制、动作联动、主循环）

建议后续拆分为：

- `vision/`：图像处理与检测逻辑
- `control/`：动作决策与机器人接口
- `config/`：阈值、舵机、相机参数配置

## 五、运行环境

- Python 3
- OpenCV (`cv2`)
- NumPy
- Hiwonder 机器人相关 SDK：
  - `hiwonder.Camera`
  - `hiwonder.Misc`
  - `hiwonder.ros_robot_controller_sdk`
  - `hiwonder.Controller`
  - `hiwonder.ActionGroupControl`

> 注：该项目依赖具体机器人硬件与官方动作组文件，建议在对应平台镜像中运行。

## 六、快速开始

1. 准备机器人硬件并确认动作组可执行（如 `stand`、`go_forward_fast` 等）。
2. 校准相机并确保标定文件存在（`CalibrationConfig` 对应路径）。
3. 检查 LAB 阈值配置与舵机配置 YAML。
4. 运行：

```bash
python3 VisualPatrol224.py
```

5. 通过窗口观察识别结果，按 `ESC` 退出。

## 七、可视化与交互

- 实时显示巡线轮廓、中心点与检测线段
- 左键可拾色（动态更新颜色目标）
- 右键可重置拾色并恢复默认状态

- 完整闭环：从视觉感知到运动控制的实时闭环实现

- 结构清晰：初始化、状态机、处理线程、主循环职责明确
